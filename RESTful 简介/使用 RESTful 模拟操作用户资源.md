需求分析：使用 RESTful 模拟用户资源的增删改查，通过路径中的占位符传递请求参数，通过不同的请求方式对应资源的不同操作

| RESTful 路径 | 请求方式     | 操作           |
| ---------- | -------- | ------------ |
| `/user`    | `GET`    | 查询所有用户信息     |
| `/user/1`  | `GET`    | 根据用户ID查询用户信息 |
| `/user`    | `POST`   | 添加用户信息       |
| `/user/1`  | `DELETE` | 根据用户ID删除用户信息 |
| `/user`    | `PUT`    | 修改用户信息       |
